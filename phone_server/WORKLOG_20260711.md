## 핵심 아키텍처

```
갤럭시 노트10 (Termux, 상시 전원)
  ├─ cron: 예측·watchdog·뉴스·날씨 (평일/매일 자동)
  ├─ cron: 대시보드(8080)·Drop(8090) 10분마다 생존확인·자동재기동
  ├─ termux-wake-lock: 화면 꺼져도 유지
  └─ Tailscale: 집 밖에서도 접근
  ↕ GitHub (waterfirst/kospi-mw-rc-research)
PC = 개발/배포용 (평소엔 꺼둬도 됨)
```

**요지:** 폰이 상시 서버, PC는 꺼도 됨. 미니PC 구매를 폰으로 대체.

## 오늘 해결한 문제 (상세는 `TROUBLESHOOTING.md`)

1. 대시보드 계속 에러 → **Termux:API 앱(APK) 미설치**가 원인 (CLI와 별개)
2. 브라우저 센서 API 실패 → HTTPS 필수 제약 → **서버측 termux-sensor**로 우회
3. `pip install adb` 빌드 실패 → `pkg install android-tools`가 정답
4. 날씨 카드 이모지 깨짐 → **PIL 벡터 아이콘**으로 폰트 무관 처리
5. "텔레그램 안 옴" → `git pull` 미완료 (옛 스크립트 실행)
6. URL 카메라 안 됨 → 카메라는 정상, **server.py가 죽어 있었음**
7. 노트북 끄면 대시보드 죽음 → `&` 대신 **setsid로 세션 분리**
8. Tailscale CLI `SIGSYS` 크래시 → 안드로이드 seccomp의 faccessat2 차단 → **Play스토어 앱**으로 전환

## 배포 방법 (폰에서)

```bash
cd ~/kospi-mw-rc-research && git pull
bash phone_server/kospi_setup.sh    # 전체 cron(예측·날씨·대시보드·Drop) 재등록
```

개별 실행:
```bash
python phone_server/weather_card.py --telegram   # 날씨 카드 테스트
python phone_server/drop.py 8090 &               # Drop 서비스
```
