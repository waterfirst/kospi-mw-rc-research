# 낙초의 3계층 배포 아키텍처

혼자 쓰는 개인 인프라를 3개 층으로 나눠 각자 잘하는 걸 맡긴다.

```
┌─────────────────────────────────────────────────────────────┐
│ 1. GitHub — 코드 관리 · 정적 호스팅                          │
│    · 모든 코드의 단일 소스 (버전관리)                        │
│    · GitHub Pages: 정적 HTML 사이트                          │
│    · 폰/서버가 여기서 pull → 배포 (네트워크 설정 불필요)     │
└───────────────┬─────────────────────────────────────────────┘
                │ git push / pull
   ┌────────────┴────────────┐
   ▼                         ▼
┌────────────────────┐   ┌────────────────────────────────────┐
│ 2. 우분투 노트북    │   │ 3. 노트10 폰서버 (Termux)          │
│    (상시 ON)        │   │    (상시 ON, 배터리 내장=UPS)      │
│                    │   │                                    │
│ · Streamlit/React  │   │ · 간단한 HTML 호스팅 (pageserve)   │
│   등 동적 서비스   │   │ · 텔레그램 메시지 전달             │
│ · 텔레그램 봇      │   │ · 폰 센서 활용 홈케어:            │
│   (Codex, Claude)  │   │   카메라 / 소리 / 자이로 / 조도    │
│ · 무거운 연산·빌드 │   │ · KOSPI 모니터·날씨·Drop          │
│ · Claude Code 상주 │   │ · 저전력, 정전에도 버팀            │
└────────────────────┘   └────────────────────────────────────┘
```

## 역할 분담 원칙

| 층 | 잘하는 것 | 안 맡기는 것 |
|----|-----------|-------------|
| **GitHub** | 버전관리, 정적 페이지, 배포 소스 | 동적 처리·상태 |
| **우분투 노트북** | 동적 웹(Streamlit/React), 무거운 봇/연산, Claude Code | 24h 초저전력 상주 |
| **노트10 폰** | 초저전력 상시 서비스, 센서 기반 홈케어, 알림 전달 | 무거운 빌드·동적 렌더 |

## 배포 흐름

### 정적 페이지 (예: 여행 계획)
```
코드작성 → GitHub push → 폰 sync.sh(5분 cron) pull → pageserve(8095) 서빙
```
네트워크 직접 연결 불필요. Claude가 push하면 폰에 자동 반영.

### 동적 서비스
```
우분투에서 Streamlit/React 실행 → (집 밖 접근은 Tailscale)
```

### 센서 홈케어 (폰)
```
카메라(CCTV·타임랩스) · 마이크(소음감지) · 자이로(움직임) · 조도(재실감지)
  → 이벤트 발생 시 텔레그램 알림
```

## 네트워크 요약 (실측)

| 경로 | 상태 | 메모 |
|------|------|------|
| GitHub ↔ 모두 | ✅ | pull/push 항상 됨 |
| 폰 로컬 (192.168.0.43) | ✅ | 같은 WiFi에서 대시보드/페이지 |
| 우분투 → 폰 직접 | ❌ | 다른 서브넷 + 안드로이드 Tailscale inbound 제약 |
| **GitHub 경유 배포** | ✅ | 그래서 이 경로가 메인 (sync.sh) |
| Tailscale (100.71.229.101) | △ | ping OK, inbound TCP는 앱 배터리설정 필요 |
